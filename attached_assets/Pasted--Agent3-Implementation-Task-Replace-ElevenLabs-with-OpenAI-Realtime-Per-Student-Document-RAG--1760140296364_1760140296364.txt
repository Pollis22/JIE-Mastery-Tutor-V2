## Goal
Replace the current ElevenLabs ConvAI system with **OpenAI Realtime API + Azure TTS/STT** to enable real-time voice tutoring that can intelligently retrieve and cite from each student's uploaded documents during live sessions. Maintain ElevenLabs-quality voice while adding per-student document awareness.

## Requirements

### 1. Voice Stack Migration
- **Remove dependency on:** ElevenLabs ConvAI agents
- **Implement:** OpenAI Realtime API for conversational AI
- **Add:** Azure Speech Services for TTS/STT (higher quality, lower cost than OpenAI's default)
- **Maintain:** Same voice quality and conversational flow as current ElevenLabs setup
- **Keep:** Grade-level appropriate teaching (K-2, 3-5, 6-8, 9-12, College) through system prompts

### 2. Document RAG Pipeline
Build a complete Retrieval-Augmented Generation system that:

#### A. Document Processing (`/api/upload` enhancement)
When a student uploads a document:
1. **Parse** the file into text:
   - PDFs: Use `pdf-parse` or `pdfjs-dist`
   - DOCX: Use `mammoth` or `docx-parser`
   - Images: Use OCR (Tesseract.js or Azure Computer Vision API)
2. **Chunk** the text into 400-800 token segments with 100-token overlap
3. **Generate embeddings** using OpenAI `text-embedding-3-small` (cheap, excellent quality)
4. **Store** in vector database with metadata:
   ```json
   {
     "userId": "user_123",
     "studentId": "student_456",
     "documentId": "doc_789",
     "chunkId": "chunk_001",
     "content": "text chunk here...",
     "embedding": [0.123, -0.456, ...],
     "metadata": {
       "filename": "algebra_homework.pdf",
       "subject": "Math",
       "gradeLevel": "9-12",
       "pageNumber": 3,
       "uploadedAt": "2025-10-10T12:00:00Z"
     }
   }
   ```

#### B. Vector Database Setup
- **Use:** PostgreSQL with `pgvector` extension (already available on Railway)
- **Create table:** `document_embeddings` with vector similarity search
- **Index:** Use HNSW or IVFFlat index for fast similarity search
- **Isolation:** CRITICAL - ensure each student can ONLY search their own documents

#### C. Retrieval Controller
During voice sessions:
1. **On each user utterance** from STT, run vector search:
   ```typescript
   const relevantChunks = await vectorSearch({
     userId: session.userId,
     studentId: session.studentId,
     query: userText,
     topK: 6,
     similarityThreshold: 0.7
   });
   ```
2. **Format context** with citations:
   ```
   RETRIEVED CONTEXT:
   [Source: algebra_homework.pdf, Page 3]
   "The quadratic formula is x = (-b ± √(b²-4ac)) / 2a..."
   
   [Source: study_guide.docx, Section 2]
   "Remember to always check your work by substituting back..."
   ```
3. **Inject into LLM** via function/tool calling or system message
4. **Enforce citation requirement** in system prompt

**Technical Implementation:**
```typescript
interface TranscriptMessage {
  id: string;
  speaker: 'student' | 'tutor';
  text: string;
  timestamp: Date;
  isComplete: boolean; // false while streaming
  citations?: Array<{
    documentId: string;
    documentName: string;
    pageNumber: number;
    startIndex: number; // where citation appears in text
    endIndex: number;
  }>;
}

const LiveTranscriptComponent = () => {
  const [messages, setMessages] = useState<TranscriptMessage[]>([]);
  const [currentStreaming, setCurrentStreaming] = useState<string>('');
  const transcriptRef = useRef<HTMLDivElement>(null);
  
  // Auto-scroll to bottom when new messages arrive
  useEffect(() => {
    if (transcriptRef.current) {
      transcriptRef.current.scrollTop = transcriptRef.current.scrollHeight;
    }
  }, [messages, currentStreaming]);
  
  return (
    <div className="live-transcript" ref={transcriptRef}>
      {messages.map((msg) => (
        <TranscriptMessage key={msg.id} message={msg} />
      ))}
      {currentStreaming && (
        <StreamingMessage text={currentStreaming} speaker="tutor" />
      )}
    </div>
  );
};
```

### 5. Database Schema Additions

Create new tables/columns:

```sql
-- Vector embeddings table
CREATE TABLE document_embeddings (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  student_id INTEGER REFERENCES students(id) ON DELETE CASCADE,
  document_id INTEGER REFERENCES documents(id) ON DELETE CASCADE,
  chunk_index INTEGER NOT NULL,
  content TEXT NOT NULL,
  embedding vector(1536), -- OpenAI embedding size
  metadata JSONB,
  created_at TIMESTAMP DEFAULT NOW()
);

-- Add pgvector index for fast similarity search
CREATE INDEX ON document_embeddings 
USING ivfflat (embedding vector_cosine_ops)
WITH (lists = 100);

-- Update sessions table to track RAG usage
ALTER TABLE sessions ADD COLUMN chunks_retrieved INTEGER DEFAULT 0;
ALTER TABLE sessions ADD COLUMN citations JSONB;
ALTER TABLE sessions ADD COLUMN transcript JSONB;

-- Add voice_stack column to track which system was used
ALTER TABLE sessions ADD COLUMN voice_stack VARCHAR(50) DEFAULT 'realtime';
```
