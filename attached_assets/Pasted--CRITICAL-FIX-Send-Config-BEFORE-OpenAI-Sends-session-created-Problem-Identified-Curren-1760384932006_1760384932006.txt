# CRITICAL FIX: Send Config BEFORE OpenAI Sends session.created

## Problem Identified

**Current Log Sequence (WRONG):**
```
1. [RealtimeWS] Connected to OpenAI ← WebSocket opens
2. [RealtimeWS] Session created confirmation ← OpenAI sends this IMMEDIATELY
3. [RealtimeWS] Built instructions for undefined (272 chars) ← Built AFTER
4. [RealtimeWS] Sending COMPLETE session config ← Sent AFTER session.created
```

**Root Cause:** When WebSocket opens, OpenAI immediately sends `session.created` with DEFAULT config (no instructions). By the time your code builds and sends instructions, OpenAI has already processed the session with empty config.

**Why This Happens:** Async code in `on('open')` handler runs AFTER OpenAI's immediate response.

---

## Solution: Build Instructions BEFORE Opening WebSocket

### Current Approach (WRONG):

```typescript
// ❌ BAD - Building instructions after connection opens

  // WebSocket is already open, OpenAI already sent session.created
  
  const instructions = await buildInstructions(...); // Too late!
  
  ws.send(JSON.stringify({
    type: 'session.update',
    session: { instructions }
  }));
});
```

### Correct Approach (FIXED):

```typescript
// ✅ GOOD - Build instructions BEFORE opening connection
async function startRealtimeSession(studentName, gradeLevel, subject) {
  
  // 1. BUILD INSTRUCTIONS FIRST (before WebSocket)
  const instructions = buildSystemInstructions(
    studentName || 'Student',
    gradeLevel || 'General',
    subject || 'General'
  );
  
  console.log(`✅ Built instructions BEFORE connection (${instructions.length} chars)`);
  
  const voice = getVoiceForGrade(gradeLevel);
  
  // 2. Prepare COMPLETE config
  const sessionConfig = {
    type: 'session.update',
    session: {
      voice: voice,
      instructions: instructions,
      modalities: ['text', 'audio'],
      temperature: 0.8,
      turn_detection: {
        type: 'server_vad',
        threshold: 0.5,
        prefix_padding_ms: 300,
        silence_duration_ms: 500
      }
    }
  };
  
  // 3. NOW open WebSocket
  const ws = new WebSocket(
    `wss://api.openai.com/v1/realtime?model=gpt-4o-realtime-preview-2024-10-01`,
    {
      headers: {
        'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
        'OpenAI-Beta': 'realtime=v1'
      }
    }
  );
  
  // 4. Send config IMMEDIATELY on open (synchronously!)
  ws.on('open', () => {
    console.log('✅ WebSocket connected, sending config IMMEDIATELY');
    
    // Send pre-built config as FIRST message
    ws.send(JSON.stringify(sessionConfig));
    
    console.log('✅ Config sent as FIRST message');
  });
  
  // 5. Handle messages
  ws.on('message', (data) => {
    const message = JSON.parse(data.toString());
    
    if (message.type === 'session.created') {
      console.log('✅ Session created with our config');
    }
    
    if (message.type === 'session.updated') {
      console.log('✅ Session updated successfully');
    }
    
    if (message.type === 'error') {
      console.error('❌ OpenAI error:', message.error);
    }
    
    // Handle other message types...
  });
  
  return ws;
}
```

## Complete Working Example

```typescript
// Backend WebSocket Handler
export async function createRealtimeSession(req, res) {
  try {
    const { studentName, gradeLevel, subject } = req.body;
    
    // STEP 1: Build config BEFORE WebSocket (no async!)
    const instructions = buildSystemInstructions(
      studentName || 'Student',
      gradeLevel || 'General', 
      subject || 'General'
    );
    
    const voice = getVoiceForGrade(gradeLevel);
    
    console.log(`✅ Instructions ready: ${instructions.length} chars`);
    
    // STEP 2: Prepare complete session config
    const sessionConfig = {
      type: 'session.update',
      session: {
        voice: voice,
        instructions: instructions,
        modalities: ['text', 'audio'],
        temperature: 0.8,
        max_response_output_tokens: 4096,
        turn_detection: {
          type: 'server_vad',
          threshold: 0.5,
          prefix_padding_ms: 300,
          silence_duration_ms: 500
        }
      }
    };
    
    // STEP 3: Create WebSocket with config ready
    const ws = new WebSocket(
      `wss://api.openai.com/v1/realtime?model=gpt-4o-realtime-preview-2024-10-01`,
      {
        headers: {
          'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
          'OpenAI-Beta': 'realtime=v1'
        }
      }
    );
    
    // STEP 4: Send config IMMEDIATELY on open
    ws.on('open', () => {
      console.log('✅ Connected, sending config NOW');
      ws.send(JSON.stringify(sessionConfig)); // Synchronous!
      console.log('✅ Config sent');
    });
    
    // STEP 5: Handle responses
    ws.on('message', (data) => {
      const msg = JSON.parse(data.toString());
      console.log('Received:', msg.type);
      
      switch (msg.type) {
        case 'session.created':
          console.log('✅ Session created:', msg.session.id);
          break;
        case 'session.updated':
          console.log('✅ Session updated with our config');
          break;
        case 'error':
          console.error('❌ Error:', msg.error);
          break;
      }
    });
    
    ws.on('error', (error) => {
      console.error('❌ WebSocket error:', error);
    });
    
    // Return success
    res.json({ success: true, sessionId: 'session_123' });
    
  } catch (error) {
    console.error('Session creation failed:', error);
    res.status(500).json({ error: error.message });
  }
}

// Synchronous instruction builder
function buildSystemInstructions(name, grade, subject) {
  const instructions = `You are an expert ${grade} tutor for ${subject}, speaking with ${name}.

PERSONALITY:
- Warm, encouraging, and patient
- Age-appropriate language for ${grade} students
- Use Socratic method - guide students to answers
- Celebrate effort and progress

TEACHING APPROACH:
1. Break complex topics into smaller steps
2. Use real-world examples for ${grade} level
3. Check understanding frequently
4. Offer hints before direct answers
5. Keep responses concise (30-60 seconds)

CONVERSATION STYLE:
- Natural and conversational
- Enthusiastic and supportive
- Adapt based on responses
- Ask follow-up questions

Goal: Help ${name} understand deeply, not just get answers.`;

  // Validate
  if (!instructions || instructions.length < 200) {
    throw new Error('Instructions not built correctly');
  }
  
  return instructions;
}

function getVoiceForGrade(grade) {
  const voices = {
    'K-2': 'alloy',
    '3-5': 'echo',
    '6-8': 'fable',
    '9-12': 'onyx',
    'College': 'nova',
    'General': 'echo'
  };
  return voices[grade] || 'echo';
}
```

## Why This Works

**OpenAI Realtime API behavior:**
1. Client opens WebSocket
2. OpenAI IMMEDIATELY sends `session.created` with default config
3. Client must send `session.update` to override defaults
4. If client sends update AFTER session.created, config may not apply correctly

**The fix:**
- Build config BEFORE opening WebSocket
- Send IMMEDIATELY on `open` event
- OpenAI receives config before processing session
- Session starts with correct instructions
