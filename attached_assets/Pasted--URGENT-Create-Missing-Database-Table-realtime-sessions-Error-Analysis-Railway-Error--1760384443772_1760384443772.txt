# URGENT: Create Missing Database Table - realtime_sessions

## Error Analysis

**Railway Error:**
```
relation "realtime_sessions" does not exist at character 13
POST /api/session/realtime/start 500
```

**Root Cause:** The `realtime_sessions` table was never created in the PostgreSQL database. Code is trying to insert session data into a non-existent table.

---

## Fix #1: Add Table Definition to Schema

### Location: `server/db/schema.ts`

Add this table definition:

```typescript
import { pgTable, serial, integer, text, timestamp, jsonb, boolean } from "drizzle-orm/pg-core";

// Existing tables (users, students, subscriptions, documents, sessions)...

// ADD THIS: Realtime voice sessions table
export const realtimeSessions = pgTable("realtime_sessions", {
  id: serial("id").primaryKey(),
  userId: integer("user_id")
    .references(() => users.id, { onDelete: 'cascade' })
    .notNull(),
  studentId: integer("student_id")
    .references(() => students.id, { onDelete: 'cascade' }),
  subject: text("subject"),
  gradeLevel: text("grade_level"),
  language: text("language").default("en"),
  voice: text("voice"), // OpenAI voice name (alloy, echo, fable, onyx, nova)
  model: text("model").default("gpt-4o-realtime-preview"),
  status: text("status").default("active"), // active, ended, error
  transcript: jsonb("transcript").default([]),
  contentDocuments: jsonb("content_documents").default([]), // Document IDs referenced
  startedAt: timestamp("started_at").defaultNow(),
  endedAt: timestamp("ended_at"),
  minutesUsed: integer("minutes_used").default(0),
  errorMessage: text("error_message"),
  createdAt: timestamp("created_at").defaultNow(),
  updatedAt: timestamp("updated_at").defaultNow()
});
```

Connect to Railway PostgreSQL and run:

```sql
CREATE TABLE IF NOT EXISTS realtime_sessions (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  student_id INTEGER REFERENCES students(id) ON DELETE CASCADE,
  subject TEXT,
  grade_level TEXT,
  language TEXT DEFAULT 'en',
  voice TEXT,
  model TEXT DEFAULT 'gpt-4o-realtime-preview',
  status TEXT DEFAULT 'active',
  transcript JSONB DEFAULT '[]'::jsonb,
  content_documents JSONB DEFAULT '[]'::jsonb,
  started_at TIMESTAMP DEFAULT NOW(),
  ended_at TIMESTAMP,
  minutes_used INTEGER DEFAULT 0,
  error_message TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Create index for faster queries
CREATE INDEX idx_realtime_sessions_user_id ON realtime_sessions(user_id);
CREATE INDEX idx_realtime_sessions_student_id ON realtime_sessions(student_id);
CREATE INDEX idx_realtime_sessions_status ON realtime_sessions(status);
```

## Troubleshooting

### Issue: Migration Fails

**Check:**
1. Are you connected to the correct database?
2. Do you have write permissions?
3. Is the database URL correct in .env?

**Solution:**
```bash
# Verify database connection
echo $DATABASE_URL

# Test connection
psql $DATABASE_URL -c "SELECT version();"
```

### Issue: Table Already Exists Error

**If you see:** `table "realtime_sessions" already exists`

**Solution:** Table is already there, just update the schema file and restart server.

### Issue: Foreign Key Constraint Fails

**Check:** Do `users` and `students` tables exist?

**Solution:**
```sql
-- Verify parent tables exist
SELECT table_name FROM information_schema.tables 
WHERE table_name IN ('users', 'students');
```

If missing, run full migration to create all tables.
