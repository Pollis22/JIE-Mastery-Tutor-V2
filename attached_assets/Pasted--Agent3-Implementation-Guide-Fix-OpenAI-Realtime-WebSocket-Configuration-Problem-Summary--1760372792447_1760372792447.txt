# Agent3 Implementation Guide: Fix OpenAI Realtime WebSocket Configuration

## Problem Summary

**Issue:** OpenAI Realtime API returns `server_error` because instructions are not included in the initial WebSocket configuration message.

**Root Cause:** When using WebSocket connection to OpenAI Realtime API, the FIRST message must contain the COMPLETE session configuration (voice + instructions + modalities). Sending voice first, then instructions as a separate update doesn't work.

---

## OpenAI Realtime WebSocket Protocol

### How it Works:
1. Client opens WebSocket connection to `wss://api.openai.com/v1/realtime`
2. **First message** sent MUST be `session.update` with COMPLETE config
3. OpenAI validates and processes the complete config
4. Audio streams can then flow bidirectionally

### Critical Rule:
**The first `session.update` message must include ALL configuration:**
- `voice` (required)
- `instructions` (required for proper behavior)
- `modalities` (optional, defaults to ['text', 'audio'])
- `temperature` (optional)
- `turn_detection` (optional)
- All other session parameters you want to set

**You CANNOT:**
- Send voice config first
- Then send instructions as a second update
- OpenAI will reject incomplete initial config with `server_error`

## Implementation Fix

### Location: `server/realtime-server.ts`

### Current Code (WRONG):

```typescript
// ❌ BAD - Sending voice first, instructions later
ws.on('connection', (socket) => {
  console.log('WebSocket connected');
  
  // First update: only voice
  socket.send(JSON.stringify({
    type: 'session.update',
    session: {
      voice: 'echo'
    }
  }));
  
  // Later update: instructions (TOO LATE!)
  const instructions = buildInstructions(studentName, grade, subject);
  socket.send(JSON.stringify({
    type: 'session.update',
    session: {
      instructions: instructions
    }
  }));
});
```

### Fixed Code (CORRECT):

```typescript
// ✅ GOOD - Sending complete config as FIRST message
ws.on('connection', (socket) => {
  console.log('✅ WebSocket connected to OpenAI Realtime');
  
  // Get session context from connection
  const { studentName, gradeLevel, subject } = socket.handshake.query;
  
  // BUILD INSTRUCTIONS FIRST (before sending anything)
  const instructions = buildSystemInstructions(
    studentName || 'Student',
    gradeLevel || 'General',
    subject || 'General'
  );
  
  console.log(`✅ Built instructions for ${studentName} (${instructions.length} chars)`);
  
  // Select appropriate voice
  const voice = getVoiceForGrade(gradeLevel);
  
  console.log(`✅ Sending COMPLETE session config as FIRST message`);
  
  // Send COMPLETE configuration as FIRST message (ONE message with everything!)
  socket.send(JSON.stringify({
    type: 'session.update',
    session: {
      // Voice configuration
      voice: voice,
      
      // System instructions (CRITICAL - must be here!)
      instructions: instructions,
      
      // Audio configuration
      modalities: ['text', 'audio'],
      
      // Response behavior
      temperature: 0.8,
      max_response_output_tokens: 4096,
      
      // Turn detection (optional but recommended)
      turn_detection: {
        type: 'server_vad',
        threshold: 0.5,
        prefix_padding_ms: 300,
        silence_duration_ms: 500
      },
      
      // Input/output audio format
      input_audio_format: 'pcm16',
      output_audio_format: 'pcm16',
      input_audio_transcription: {
        model: 'whisper-1'
      }
    }
  }));
  
  console.log('✅ Complete session configuration sent');
  
  // Now set up message handlers for ongoing conversation
  socket.on('message', (data) => {
    try {
      const message = JSON.parse(data.toString());
      handleRealtimeMessage(socket, message);
    } catch (error) {
      console.error('❌ Error parsing message:', error);
    }
  });
  
  socket.on('close', () => {
    console.log('WebSocket disconnected');
    // Clean up resources
  });
  
  socket.on('error', (error) => {
    console.error('❌ WebSocket error:', error);
  });
});

// Instruction builder (synchronous)
function buildSystemInstructions(
  studentName: string,
  gradeLevel: string,
  subject: string
): string {
  const instructions = `You are an expert ${gradeLevel} tutor for ${subject}, having a voice conversation with ${studentName}.

PERSONALITY:
- Warm, encouraging, and patient
- Use age-appropriate language for ${gradeLevel} students
- Apply the Socratic method - guide students to discover answers
- Celebrate effort: "Great thinking!", "You're on the right track!"

TEACHING APPROACH:
1. Break complex topics into smaller, digestible steps appropriate for ${gradeLevel}
2. Use real-world examples and analogies that resonate with this age group
3. Check understanding frequently with simple questions
4. If student is stuck, offer hints before giving direct answers
5. Keep responses concise for voice (30-60 seconds per turn)

CONVERSATION STYLE:
- Natural and conversational, not robotic or overly formal
- Enthusiastic and supportive tone
- Adapt difficulty based on student's responses
- Ask follow-up questions to ensure comprehension
- Use encouragement liberally

IMPORTANT: You are in a real-time voice conversation. Speak naturally as if you're sitting with ${studentName} in person.

Your goal: Help ${studentName} understand concepts deeply, not just memorize answers.`;

  // Validate
  if (!instructions || instructions.length < 200) {
    throw new Error('Instructions not built correctly');
  }
  
  if (!instructions.includes(studentName)) {
    throw new Error('Instructions must include student name');
  }
  
  return instructions;
}

// Voice selection based on grade
function getVoiceForGrade(gradeLevel: string): string {
  const voiceMap: Record<string, string> = {
    'K-2': 'alloy',      // Clear, friendly for young kids
    '3-5': 'echo',       // Upbeat, engaging for middle elementary
    '6-8': 'fable',      // Warm, expressive for middle school
    '9-12': 'onyx',      // Deep, authoritative for high school
    'College': 'nova',   // Professional, bright for college
    'Adult': 'nova',     // Professional for adults
    'General': 'echo'    // Default
  };
  
  return voiceMap[gradeLevel] || 'echo';
}

// Handle incoming messages from OpenAI
function handleRealtimeMessage(socket: WebSocket, message: any) {
  console.log('Received OpenAI message:', message.type);
  
  switch (message.type) {
    case 'session.created':
      console.log('✅ Session created:', message.session.id);
      break;
      
    case 'session.updated':
      console.log('✅ Session updated successfully');
      break;
      
    case 'error':
      console.error('❌ OpenAI error:', message.error);
      
      if (message.error.type === 'server_error') {
        console.error('❌ Server error - likely configuration issue');
        console.error('Error details:', message.error.message);
      }
      
      // Forward error to client
      socket.send(JSON.stringify({
        type: 'error',
        error: message.error
      }));
      break;
      
    case 'conversation.item.created':
      if (message.item.role === 'user') {
        // User spoke, transcription available
        console.log('👤 User:', message.item.content?.[0]?.transcript);
      }
      break;
      
    case 'response.audio_transcript.delta':
      // AI is speaking, transcript chunk
      console.log('🤖 AI (partial):', message.delta);
      break;
      
    case 'response.audio_transcript.done':
      // AI finished speaking
      console.log('🤖 AI (complete):', message.transcript);
      break;
      
    case 'response.done':
      console.log('✅ Response completed');
      break;
      
    default:
      // Forward other messages to client
      socket.send(JSON.stringify(message));
  }
}
```

## Common Mistakes to Avoid

❌ **Sending multiple session.update messages** - First one must be complete  
❌ **Building instructions asynchronously** - Must be synchronous and complete before first message  
❌ **Not including instructions in first message** - Will cause server_error  
❌ **Sending voice config first, instructions later** - OpenAI processes first message immediately  
❌ **Empty or null instructions** - Validate before sending  
❌ **Wrong voice name** - Must be one of: alloy, echo, fable, onyx, nova, shimmer  
❌ **Missing model parameter in URL** - Must include model version
