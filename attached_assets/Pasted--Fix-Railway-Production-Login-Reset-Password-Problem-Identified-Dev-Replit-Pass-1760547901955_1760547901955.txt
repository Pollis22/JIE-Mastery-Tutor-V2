# Fix Railway Production Login - Reset Password

## Problem Identified

✅ **Dev (Replit):** Password reset successful, login works  
❌ **Railway (Production):** Password still wrong, login fails

**Root Cause:** Password was reset in dev database only. Railway uses a separate PostgreSQL database that still has the old/wrong password.

---

## Solution: Reset Password on Railway Database

### Option 1: Quick Fix via Railway CLI (Recommended)

**Step 1: Generate the password hash**

Run this in your **local terminal or Replit console**:

```bash
node -e "
const crypto = require('crypto');
const { promisify } = require('util');
const scrypt = promisify(crypto.scrypt);

(async () => {
  const password = 'Crenshaw22\$\$';
  const salt = crypto.randomBytes(16).toString('hex');
  const buf = await scrypt(password, salt, 64);
  const hash = buf.toString('hex') + '.' + salt;
  console.log('Your password hash:');
  console.log(hash);
})();
"
```

**This outputs something like:**
```
Your password hash:
a3f7b2c1d4e5f6...abc123.def456789...
```

**Step 2: Update password in Railway database**

```bash

"
```

### Option 2: Use Script on Railway (Alternative)

**Create this script: `server/scripts/reset-railway-password.ts`**

```typescript
import { scrypt, randomBytes } from 'crypto';
import { promisify } from 'util';
import pkg from 'pg';
const { Client } = pkg;

const scryptAsync = promisify(scrypt);

async function hashPassword(password: string): Promise<string> {
  const salt = randomBytes(16).toString('hex');
  const buf = (await scryptAsync(password, salt, 64)) as Buffer;
  return `${buf.toString('hex')}.${salt}`;
}

async function resetPasswordOnRailway() {
  const email = 'pollis@mfhfoods.com';
  const newPassword = 'Crenshaw22$$';
  
  console.log('🔐 Resetting password on Railway database...');
  console.log('Email:', email);
  
  // Connect to Railway PostgreSQL
  const client = new Client({
    connectionString: process.env.DATABASE_URL,
    ssl: {
      rejectUnauthorized: false
    }
  });
  
  try {
    await client.connect();
    console.log('✅ Connected to Railway database');
    
    // Hash the password
    const hashedPassword = await hashPassword(newPassword);
    console.log('✅ Password hashed');
    
    // Update the password
    const result = await client.query(`
      UPDATE users 
      SET password = $1
      WHERE email = $2
      RETURNING id, email, username
    `, [hashedPassword, email]);
    
    if (result.rows.length === 0) {
      console.error('❌ User not found:', email);
      process.exit(1);
    }
    
    const user = result.rows[0];
    console.log('✅ Password updated successfully!');
    console.log('');
    console.log('═══════════════════════════════════════');
    console.log('🎉 LOGIN CREDENTIALS FOR RAILWAY:');
    console.log('═══════════════════════════════════════');
    console.log('Email:', email);
    console.log('Password:', newPassword);
    console.log('User ID:', user.id);
    console.log('Username:', user.username);
    console.log('═══════════════════════════════════════');
    console.log('');
    console.log('Test at: https://jie-mastery-tutor-v2-production.up.railway.app/auth');
    
  } catch (error) {
    console.error('❌ Error:', error);
    throw error;
  } finally {
    await client.end();
  }
}

resetPasswordOnRailway()
  .then(() => {
    console.log('✅ Script completed successfully');
    process.exit(0);
  })
  .catch((error) => {
    console.error('❌ Script failed:', error);
    process.exit(1);
  });
```

**Run it targeting Railway:**

```bash
railway run tsx server/scripts/reset-railway-password.ts
```

### If Railway CLI isn't installed:
```bash
# Install Railway CLI
npm i -g @railway/cli

# Login
railway login

# Link to your project
railway link
```

### If hash generation fails:
Use this online tool: https://bcrypt.tools/ (choose 10 rounds)
- Enter: `Crenshaw22$$`
- Copy the bcrypt hash
- **BUT:** Your app uses scrypt, not bcrypt, so use the Node.js command above

### If SQL update fails:
Check the exact column name:
```bash
railway run psql $DATABASE_URL -c "\d users"
```

Look for the password column - it should be `password` (lowercase).
