# URGENT FIX: White Popup Box with Invisible Text

## Problem
There is a white popup/toast notification appearing in the bottom right corner of the screen with **white text on white background**, making it completely unreadable.

## Location
Bottom right corner of the screen (see screenshot - appears on auth page and likely throughout the app)

## Root Cause
A toast notification or alert component has:
- White or very light background
- White or very light text color
- No proper contrast

## Required Fix

### Step 1: Find the Toast/Notification Component
Search the codebase for:
- `toast` components (likely using shadcn/ui Toast or a custom notification system)
- `alert` components
- `notification` components
- Check `client/src/components/ui/` folder for toast.tsx or similar
- Check for any global notification/alert wrappers in the root layout

### Step 2: Apply High-Contrast Styling

**For Error/Warning Toasts:**
```css
.toast-error,
.alert-error,
[data-type="error"] {
  background-color: #DC2626 !important; /* Red-600 */
  color: #FFFFFF !important; /* White text */
  font-weight: 600 !important;
  border: 2px solid #991B1B !important; /* Red-800 border */
}
```

**For Success Toasts:**
```css
.toast-success,
.alert-success,
[data-type="success"] {
  background-color: #16A34A !important; /* Green-600 */
  color: #FFFFFF !important; /* White text */
  font-weight: 600 !important;
  border: 2px solid #166534 !important; /* Green-800 border */
}
```

**For Info/Default Toasts:**
```css
.toast-info,
.alert-info,
[data-type="info"],
.toast-default {
  background-color: #1E293B !important; /* Slate-800 */
  color: #FFFFFF !important; /* White text */
  font-weight: 600 !important;
  border: 2px solid #0F172A !important; /* Slate-900 border */
}
```
