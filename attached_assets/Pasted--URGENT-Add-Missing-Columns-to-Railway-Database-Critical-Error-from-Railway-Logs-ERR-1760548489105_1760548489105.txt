# URGENT: Add Missing Columns to Railway Database

## Critical Error from Railway Logs

```
ERROR: column "subscription_minutes_used" does not exist at character 317
```

**This error is blocking ALL logins.** The code expects columns that don't exist in Railway's PostgreSQL database.

---

## One-Command Fix (Run This NOW)

Copy and paste this **entire block** into your Replit Shell:

```bash
railway run psql $DATABASE_URL << 'EOF'
-- Add all missing voice tracking columns
ALTER TABLE users 
  ADD COLUMN IF NOT EXISTS subscription_minutes_used INTEGER DEFAULT 0,
  ADD COLUMN IF NOT EXISTS subscription_minutes_limit INTEGER DEFAULT 60,
  ADD COLUMN IF NOT EXISTS purchased_minutes_balance INTEGER DEFAULT 0,
  ADD COLUMN IF NOT EXISTS billing_cycle_start TIMESTAMPTZ DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS last_reset_at TIMESTAMPTZ DEFAULT NOW();

-- Add email verification columns
ALTER TABLE users
  ADD COLUMN IF NOT EXISTS email_verified BOOLEAN DEFAULT true,
  ADD COLUMN IF NOT EXISTS email_verified_at TIMESTAMPTZ DEFAULT NOW(),
  ADD COLUMN IF NOT EXISTS email_verification_token TEXT,
  ADD COLUMN IF NOT EXISTS email_verification_expires TIMESTAMPTZ;

-- Add password reset columns
ALTER TABLE users
  ADD COLUMN IF NOT EXISTS password_reset_token TEXT,
  ADD COLUMN IF NOT EXISTS password_reset_expires TIMESTAMPTZ;

-- Mark all existing users as verified (they registered before verification was required)
UPDATE users 
SET email_verified = true, 
    email_verified_at = COALESCE(email_verified_at, created_at)
WHERE email_verified IS NULL OR email_verified = false;

-- Show what we just added
SELECT 
  'Added columns successfully!' as status,
  COUNT(*) as total_users,
  COUNT(*) FILTER (WHERE email_verified = true) as verified_users
FROM users;
EOF
```

**Expected output:**
```
ALTER TABLE
ALTER TABLE
ALTER TABLE
UPDATE 1
       status                | total_users | verified_users
-----------------------------|-------------|---------------
 Added columns successfully! |     1       |      1
```

## Why This Is Critical

The login code queries the `users` table like this:

```sql
SELECT 
  id, email, username, password,
  subscription_minutes_used,    -- ❌ Doesn't exist!
  subscription_minutes_limit,   -- ❌ Doesn't exist!

FROM users 
WHERE email = $1
```

Without these columns, **every login attempt fails** with a database error before it even checks the password.
