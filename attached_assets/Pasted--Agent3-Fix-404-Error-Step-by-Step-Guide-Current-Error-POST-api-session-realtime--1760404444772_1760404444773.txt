# Agent3: Fix 404 Error - Step by Step Guide

## Current Error

```
POST /api/session/realtime → 404 (Not Found)
Error: Failed to get credentials: 404 {"error":"Session not found"}
```

The endpoint doesn't exist yet. Let's create it properly.

---

## Step 1: Find Your Main Server File

Look for one of these files:
- `server/src/index.ts`
- `server/src/app.ts`
- `server/src/server.ts`
- `server/index.ts`

This is where you initialize Express and mount routes.

---

## Step 2: Add the Session Endpoint DIRECTLY to Main File

**Don't create a separate route file yet** - let's add it directly to see it work first.

**In your main server file (e.g., `server/src/index.ts`), add this:**

```typescript
import express from 'express';

const app = express();

// Parse JSON bodies
app.use(express.json());

// ============================================
// ADD THIS ENDPOINT DIRECTLY HERE
// ============================================
app.post('/api/session/realtime', async (req, res) => {
  console.log('🎬 [ENDPOINT HIT] POST /api/session/realtime');
  console.log('📦 Request body:', req.body);

  try {
    // Check if OpenAI API key exists
    const apiKey = process.env.OPENAI_API_KEY;
    if (!apiKey) {
      console.error('❌ OPENAI_API_KEY not found in environment');
      return res.status(500).json({ 
        error: 'Server configuration error',
        message: 'OPENAI_API_KEY not configured' 
      });
    }

    console.log('🔑 OPENAI_API_KEY found, making request to OpenAI...');

    // Request ephemeral session from OpenAI
    const openaiResponse = await fetch('https://api.openai.com/v1/realtime/sessions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        model: req.body.model || 'gpt-4o-realtime-preview',
        voice: req.body.voice || 'alloy',
      }),
    });

    console.log('📡 OpenAI response status:', openaiResponse.status);

    if (!openaiResponse.ok) {
      const errorText = await openaiResponse.text();
      console.error('❌ OpenAI API error:', errorText);
      return res.status(openaiResponse.status).json({ 
        error: 'OpenAI API error',
        details: errorText 
      });
    }

    const sessionData = await openaiResponse.json();
    console.log('✅ Session created:', {
      id: sessionData.id,
      hasClientSecret: !!sessionData.client_secret?.value
    });

    // Return the credentials
    res.json({
      success: true,
      sessionId: sessionData.id,
      client_secret: sessionData.client_secret,
      model: sessionData.model,
      expires_at: sessionData.expires_at,
    });

  } catch (error: any) {
    console.error('❌ Error in /api/session/realtime:', error);
    res.status(500).json({ 
      error: 'Internal server error',
      message: error.message 
    });
  }
});

// ============================================
// ADD TEST ENDPOINT TO VERIFY IT'S WORKING
// ============================================
app.get('/api/session/test', (req, res) => {
  res.json({ 
    message: 'Session endpoint is working!',
    timestamp: new Date().toISOString(),
    env: {
      hasOpenAIKey: !!process.env.OPENAI_API_KEY,
      nodeEnv: process.env.NODE_ENV
    }
  });
});

// Your other routes and middleware...
// ... rest of your server code

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`✅ Server running on port ${PORT}`);
  console.log(`✅ Session endpoint: POST /api/session/realtime`);
  console.log(`✅ Test endpoint: GET /api/session/test`);
});
```

## Common Issues & Fixes

### Issue 1: Still Getting 404

**Problem:** The endpoint code exists but isn't running.

**Solution:** Check which file Railway is actually running.

In Railway logs, look for the startup command. It might be:
```
node dist/server/index.js
node dist/index.js
node build/server.js
```

Make sure you edited the correct source file that compiles to that location.

**Quick check:** Add this at the very top of your server file:
```typescript
console.log('🚀 SERVER FILE LOADED - THIS IS THE RIGHT FILE!');
```

If you don't see this in Railway logs, you're editing the wrong file.

### Issue 2: OPENAI_API_KEY is undefined

**Problem:** Environment variable not set.

**Solution:** 
1. Go to Railway Dashboard → Variables
2. Add `OPENAI_API_KEY` with your key
3. Click "Deploy" to restart

### Issue 3: OpenAI returns error

**Problem:** API key invalid or doesn't have Realtime API access.

**Solution:**
1. Go to OpenAI Dashboard: https://platform.openai.com/api-keys
2. Create a new key or verify existing one
3. Make sure your account has Realtime API access
4. Update in Railway

## Alternative: Minimal Working Example

If you're still stuck, replace your ENTIRE server file with this minimal version:

```typescript
import express from 'express';
import cors from 'cors';

const app = express();

app.use(cors());
app.use(express.json());

// Health check
app.get('/health', (req, res) => {
  res.json({ status: 'ok' });
});

// Session endpoint
app.post('/api/session/realtime', async (req, res) => {
  console.log('🎬 Creating session');
  
  try {
    const response = await fetch('https://api.openai.com/v1/realtime/sessions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${process.env.OPENAI_API_KEY}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        model: 'gpt-4o-realtime-preview',
        voice: 'alloy',
      }),
    });

    const data = await response.json();
    console.log('✅ Session created');
    res.json(data);
    
  } catch (error) {
    console.error('❌ Error:', error);
    res.status(500).json({ error: error.message });
  }
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`Server on port ${PORT}`);
});
```

This is the absolute minimum needed. If this doesn't work, the problem is environmental (Railway config, build process, etc.), not the code.
